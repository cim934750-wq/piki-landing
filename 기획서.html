4 랜딩페이지 설계

정보 구조 (IA)

PIKI 전환 시스템 랜딩페이지
│
├─ [NAV] 글로벌 네비게이션 — 로고 / 메뉴(5) / CTA 버튼
├─ [HERO] 메인 비주얼 — 헤드카피 + 서브카피 + CTA x2 + 핵심지표(7일/ROAS/CAC) + 목업
├─ [PAIN] 고객 고충 — 카드 x4 (전환부재 / 링크부재 / 성과부진 / 데이터부재)
├─ [TARGET] 타겟 고객 — 카드 x4 (전환↓ / 프로필뿐 / 성과↓ / DM만)
├─ [SYSTEM] 전환 시스템 — 광고유입 → 전환랜딩 → 데이터추적 → 재전환/리타겟
├─ [SERVICE] 설치 항목 — 전환 랜딩 ★ / 데이터 추적 / SNS+리타겟
├─ [PROCESS] 설치 프로세스 — 진단 → 설계 → 구축 → 런칭
├─ [METRICS] 데이터 — 업종별 광고 효율 개선 시나리오 (카페/뷰티/SaaS/교육)
├─ [PRICING] 가격 — 스타터 / 프로 [BEST] / 그로스
├─ [GUARANTEE] 보장 — 사전 시안 / 납기 보장 / 수정 합의
├─ [FAQ] 자주 묻는 질문 — 5개 항목
├─ [CTA] 문의 폼 — 이름 / 연락처 / 상황 선택 / 광고 예산 / 메시지
└─ [FOOTER] 푸터 — 브랜드 / 링크 / 저작권

전환 전략

전략적용 위치기법기대 효과
고충 공감→해결PAIN → SERVICE문제 인식 → 솔루션 제시서비스 필요성 자각
데이터 기반 증거METRICS업종별 개선 시나리오 제시신뢰도 확보
가격 앵커링PRICING3단 구조 + BEST 강조프로 상품 유도
다중 CTANAV / HERO / CTA전 구간 전환 가능이탈 최소화
